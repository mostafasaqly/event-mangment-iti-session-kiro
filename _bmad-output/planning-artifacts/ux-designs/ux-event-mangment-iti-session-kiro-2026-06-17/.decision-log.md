## Update — UI system switched to Angular Material / MD3 (2026-06-17)

- **Reversed prior decision:** "Angular + SCSS, custom token system (no component library)" → **Angular Material (Material Design 3)**, because the user has Angular Material installed. Material is a UI system, so per the UX spine rules DESIGN.md now specifies the **brand-layer delta on top of MD3** and EXPERIENCE.md Foundation inherits from it.
- **DESIGN.md rewritten to map onto MD3 token architecture:**
  - Color: trust-blue **seed** → MD3 tonal palette via Material Theme Builder; brand sets `primary`/`primary-container`, quiet `secondary`/`tertiary`, MD3 surface tiers, `error` role (reused for destructive + Rejected).
  - 7-status palette expressed as MD3 **custom color groups** (color/container/on-container), rendered as `mat-chip` badges.
  - Type scale → MD3 roles (`display-small`, `headline-small`, `title-medium`, `body-medium`, `label-medium/small`) + custom mono role (Roboto Mono) for reference/amounts.
  - Shape → MD3 **shape scale** tightened one notch; status badges use `corner-full`.
  - Elevation → MD3 **elevation levels** (level0–5).
  - Components mapped to `mat-card`, `mat-chip`, `mat-form-field`, `mat-flat/stroked-button`, `mat-table`, `mat-sidenav`; custom tokens kept (amount-pair, method-dot, event-link-reward, internal-note).
  - RTL via Angular `dir="rtl"` + CDK BidiModule. Dark theme deferred (MD3 generates from same seed).
- **EXPERIENCE.md Foundation** updated to name Angular Material / MD3; spine now specifies only the behavioral delta on top of Material.
- Discipline note added: inherit MD3 interaction/a11y defaults; brand only via color roles, type family, shape, custom tokens. `status` stays `final`.
